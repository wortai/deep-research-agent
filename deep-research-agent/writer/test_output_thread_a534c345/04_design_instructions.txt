7. SVG & CHARTS.CSS CONSTRUCTION
a) SVG:
- Dimensions: Use viewBox="0 0 800 400" for wide span concepts (waterfalls, area projections) and viewBox="0 0 400 400" for square formats (donut allocations).
- Fonts: Use inline 'font-family="Inter, sans-serif"'. Titles at 0.875rem (14px), labels/axes at 0.6875rem (11px).
- Stroke Widths: 1px for structural background gridlines, 2px for data plots/lines, 2.5px for main axes.
- Color Application: Strict adherence to the 6-hex sequence and semantic colors. No generic color names.
- Accessibility: <title> and <desc> are absolutely mandatory, briefly detailing the trend or values depicted.
- Patterns: Use waterfall charts for return expectations minus tax/fees. Use donut charts for the $1M allocation splits.

b) Charts.css:
- Chart Types: Use 'line' for historical indices, 'column' for period-by-period returns, and 'bar' or 'stacked' for segment breakdowns.
- Base Classes: Always apply 'show-labels', 'show-data', 'show-headings'. Use 'data-after' when explicitly mapping final period targets.
- Color Sequence: Apply multi-series styling mapping exactly to the hex sequence (#3182CE, #319795, #DD6B20...).
- Values: Use raw decimals for --size variables (e.g. --size: 0.85). 
- Prohibition: Never use Charts.css for 2 data points or fewer. Never deploy without a <caption> citing the data source or calculation model.

8. WHAT TO AVOID
- Never embed critical metrics (like exactly how much a 1031 exchange saves) exclusively within prose; numbers must be elevated visually via tables, bolding, or KPI cards.
- Never deploy qualitative charts. If it is in a chart, it must represent real or calculated financial figures.
- Never mix warm and cool non-semantic colors arbitrarily; the report must look clinical and restricted to its explicit sequence.
- Never split data tables or large SVGs/charts across page breaks. Keep contextual graphics directly unified with their prose explanation.
- Never build a chart without explicit source attribution, axis labels, and data points. Anonymous/unlabeled financial data implies a lack of rigor.
